### 任务 1.3：数据库创建与 Diesel 迁移 done

**目标**：用 Diesel CLI 创建数据库和4张核心表

**前提**：PostgreSQL 服务已运行，diesel_cli 已安装

**具体操作**：

1. `psql -U postgres` → `CREATE DATABASE medical_data;`
2. `cd medical-server && diesel setup`
3. `diesel migration generate create_tables`
4. 编辑 `migrations/xxx_create_tables/up.sql`，内容来自确定版第7节，包含：
   - `update_updated_at_column()` 触发器函数
   - `body_parts` 表 + 索引 + 触发器 + 6条初始数据
   - `hospitals` 表 + 索引 + 触发器
   - `medical_records` 表 + 索引 + 触发器 + CHECK约束
   - `files` 表 + 索引 + CHECK约束（file_type IN ('image','model','annotation','report')）
5. 编辑 `migrations/xxx_create_tables/down.sql`：
   ```sql
   DROP TABLE IF EXISTS files;
   DROP TABLE IF EXISTS medical_records;
   DROP TABLE IF EXISTS hospitals;
   DROP TABLE IF EXISTS body_parts;
   DROP FUNCTION IF EXISTS update_updated_at_column();
   ```
6. `diesel migration run` 执行迁移
7. `diesel print-schema > src/schema.rs` 生成 schema

**产出文件**：

- `medical-server/migrations/xxx_create_tables/up.sql`
- `medical-server/migrations/xxx_create_tables/down.sql`
- `medical-server/src/schema.rs`（自动生成）

**验证标准**：

- `diesel migration run` 无错误
- `diesel migration redo` 可正常回滚重建
- `psql` 中 `\dt` 显示4张表，`SELECT * FROM body_parts` 返回6条数据

## Step 2：记录列表页面 Done

### 任务 2.1：记录查询服务 (record_service.rs — 列表查询)

**目标**：实现记录列表查询（分页 + JOIN 医院/部位名称）

**具体操作**：
编辑 `src/services/record_service.rs`，实现：

1. **分页参数结构体**：

   ```rust
   pub struct PaginationParams {
       pub page: i64,      // 当前页码（从1开始）
       pub per_page: i64,  // 每页条数（默认20）
   }
   ```
2. **分页结果结构体**：

   ```rust
   pub struct PaginatedRecords {
       pub records: Vec<MedicalRecordView>,
       pub total: i64,
       pub page: i64,
       pub per_page: i64,
       pub total_pages: i64,
   }
   ```
3. **列表查询函数 `list_records(pool, pagination) -> QueryResult<PaginatedRecords>`**：

   - COUNT 查询获取总数
   - `medical_records` JOIN `hospitals` JOIN `body_parts`
   - SELECT: record 字段 + `hospitals.hospital_name` + `body_parts.part_name` + `body_parts.part_code`
   - ORDER BY `medical_records.id DESC`
   - LIMIT + OFFSET 分页
   - 对每条记录查询关联文件数（子查询或单独查询）
   - 组装为 `Vec<MedicalRecordView>` 返回
4. **字典查询辅助函数**：

   - `list_active_hospitals(pool) -> QueryResult<Vec<Hospital>>` — 查询所有启用的医院（列表页筛选下拉用）
   - `list_active_body_parts(pool) -> QueryResult<Vec<BodyPart>>` — 查询所有启用的部位

**产出文件**：`medical-server/src/services/record_service.rs`（首次实质内容）

     - 构建 `record_date = NaiveDate::from_ymd_opt(year, month, day)`
     - 构建 `NewMedicalRecord` 并插入数据库，获得 `record.id`
   - **处理每个文件**：
     - 根据字段名确定 `file_type`（image_files→image, model_files→model, ...）
     - 调用 `get_full_extension()` 提取扩展名
     - 调用 `validate_file_type()` 验证
     - 调用 `generate_storage_name()` 生成存储文件名
     - 查询 `body_parts` 表获取部位的 `part_code` 和 `bucket_name`
     - 调用 `MinioClient::build_object_key()` 构建对象键
     - 调用 `minio.upload_object()` 上传到 MinIO
     - 构建 `NewFileRecord` 并插入 files 表
   - **完成**：重定向到 `/records/{id}`（记录详情页）
2. 在 `routes/mod.rs` 注册路由

**产出文件**：`medical-server/src/routes/records.rs`（追加函数）

**验证标准**：

- 能成功创建记录（数据库 `medical_records` 有新行）
