# Viewer 页面加载性能优化方案

## 优化空间分析（按影响大小排序）

---

### 一、网络层（影响最大）

#### 1. 每个模型文件都触发独立的 N+1 次 DB 查询

`api_file_data` 每次被调用都要：

- 查文件记录（1 次）
- 查 `medical_records` 拿 `body_part_id`（1 次）
- 查 `body_parts` 拿 `part_code`（1 次）

如果有 5 个模型，前端并发 5 次 = **15 次 DB 查询**，且 `record_id → body_part_id → part_code` 对同一病历每次结果完全相同。

> **优化**：`api_list_models` 返回结果中附带 `object_key` + `part_code`，然后新增一个批量接口或让前端通过一个接口一次拿所有文件数据。

#### 2. 模型文件列表和文件数据是串行的

`initialize` 先 `await fetchModelList`，拿到列表后再 `await loadAllModels`。

> **优化**：如果后端新增批量接口，可以一个请求返回所有数据；否则至少可以在拿到列表的同时就开始预热 VTK pipeline。

#### 3. 无 HTTP 缓存头

返回的模型数据没有 `Cache-Control` / `ETag` 头，每次刷新都要重新从 MinIO 拉取全部数据。

> **优化**：STL 文件不变的情况下，加 `Cache-Control: max-age=3600` 或 `ETag`，浏览器刷新时 304 即可。

---

### 二、前端加载时序（影响较大）

#### 4. VTK Pipeline 初始化与数据请求完全串行

当前时序：

```
onMounted → 创建 WebGL pipeline → initialize() → fetchModelList → fetchFileData×N → loadSTL×N
```

`fetchModelList` 是纯网络请求，不依赖 WebGL，但 `initialize()` 在 `App.vue` 的 `onMounted` 中才调用，此时已经浪费了 Vue 挂载 + WebGL 初始化的时间。

> **优化**：在模块加载时（`<script setup>` 顶层）立即发起 `fetchModelList`，与 Vue 挂载 + WebGL 初始化并行，拿到数据后再喂给 renderer。

#### 5. loadModel 每次触发 renderScene

`loadModel` 每加载一个模型就调一次 `renderScene()`，5 个模型 = 5 次渲染。最后 `resetCamera` 又渲染一次。

> **优化**：加载过程中不渲染，全部加载完 `resetCamera` 时一次性渲染。

---

### 三、VTK 处理管线（影响中等）

#### 6. 所有模型在主线程同步处理

`results.forEach` 同步依次调用 `loadSTL`。每个 `loadSTL` 包含 CleanPolyData + WindowedSincSmooth + PolyDataNormals，都是 CPU 密集操作，主线程阻塞。

> **优化**：用 `requestIdleCallback` 或 `setTimeout(0)` 在每个模型之间让出主线程，或考虑 Web Worker 做前处理（但 vtk.js 限制较多）。

#### 7. 平滑迭代次数可按模型大小自适应

固定 `numberOfIterations: 10`。对小模型没必要跑 10 次平滑。

> **优化**：根据三角面数量动态选择迭代次数（如 <10K 面可跳过平滑）。

---

### 四、构建优化（影响较小但一次性收益）

#### 8. vtk.js 没有 tree-shaking 配置

`vite.config.ts` 中没有针对 vtk.js 的 manualChunks 或 optimizeDeps 配置。vtk.js 体积庞大，默认打包可能包含未使用模块。

> **优化**：Vite 配置 `build.rollupOptions.output.manualChunks` 将 vtk.js 拆成独立 chunk，利用浏览器缓存。

---

## 优先级排序


四、模型操作
功能	函数
拖拽模式	changeSelectPart() + DragControls — 选中部件后可拖动位置
爆炸视图	changePart() — 重置所有模型位置（配合拖拽使用）

五、测量工具
功能	函数
距离测量	distancePoints() — 在模型表面点选两点，计算并标注距离（mm）
角度测量	anglePoints() — 点选三点，计算并标注夹角（°）
标签标注	labelPoint() — 点选一点后输入文字标签
删除标注	deleteItem() — 删除选中的测量/标注
病灶-器官表面测距	featureMeasureUterus() — 基于预计算的距离标记数据，显示器官表面到病灶的距离


六、模型裁切
功能	说明
三轴剖切	clippingPart() — 通过 clipPlanes（X/Y/Z 三个裁切面），配合 jRange 滑块实时剖切模型
