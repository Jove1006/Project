# 光源跟随相机示意图

下面这张图对应当前 `lights.ts` 的逻辑：
- 先从相机得到 `forward/right/up`
- 再用每盏灯的 `offset` 组合成方向
- 最后得到世界坐标位置并指向 `focalPoint`

```mermaid
flowchart LR
  subgraph Camera[相机坐标系]
    CPos[Camera Position]
    Focal[Focal Point]
    Forward[forward = normalize(focal - position)]
    Right[right = normalize(cross(forward, viewUp))]
    Up[up = normalize(cross(right, forward))]

    CPos --> Forward
    Focal --> Forward
    Forward --> Right
    Right --> Up
  end

  subgraph Offsets[三点光源 offset]
    O1[Key:  offset ( 1,  1,  1), intensity 1.0]
    O2[Fill: offset (-1, 0.5, 0), intensity 0.6]
    O3[Back: offset ( 0, -1, -1), intensity 0.3]
  end

  subgraph Mapping[从相机坐标映射到世界坐标]
    Formula[offsetVec = right*ox + up*oy + forward*oz]
    Dist[scaleFactor = max(distance*0.6, 1)]
    WPos[worldPos = focal + offsetVec*scaleFactor]
    LookAt[light.setFocalPoint(focal)]
  end

  O1 --> Formula
  O2 --> Formula
  O3 --> Formula
  Right --> Formula
  Up --> Formula
  Forward --> Formula
  Formula --> WPos
  Dist --> WPos
  Focal --> WPos
  WPos --> LookAt
```

---

## 直观理解

- 这 3 盏灯不是固定在世界某个点，而是固定在“相机周围的相对位置”。
- 当你旋转/缩放视角时，`updateFromCamera()` 会重新计算灯的位置。
- 结果是：当前看到的正面通常都会有主光和补光，不容易出现“转过来就全暗”。

---

## 与代码对应关系

- 坐标轴计算：`forward/right/up`
- 偏移变换：`offsetVec = right*ox + up*oy + forward*oz`
- 世界坐标：`worldPos = focal + offsetVec*scaleFactor`
- 光源朝向：`light.setFocalPoint(...focal)`
