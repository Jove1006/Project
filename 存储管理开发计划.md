# 医学数据存储管理系统 - 开发计划

## 1. 目标

开发一个**可用的医学数据存储和管理软件**，聚焦核心功能：
- 数据的录入、存储、查询、编辑、删除
- 文件（DICOM、STL、标注、报告）的上传、下载、管理
- 简洁直观的操作界面

**原则：先跑起来，再完善。**

---

## 2. 技术选型

| 模块 | 技术 | 选型理由 |
|------|------|----------|
| 后端框架 | Rust + Actix-web 4 | 极致性能、内存安全、编译期保障 |
| 数据库 | PostgreSQL 15+ | 企业级关系数据库，功能强大，扩展性强 |
| ORM/查询 | Diesel 2 | Rust主流ORM，编译期类型安全查询，零开销抽象 |
| 模板引擎 | Tera | Rust生态主流模板引擎，语法类似Jinja2 |
| 文件存储 | MinIO | S3兼容对象存储，可私有部署，高性能，易扩展 |
| 前端 | HTML + CSS + JavaScript | Tera模板 + Bootstrap 5，无需前端构建工具 |
| 序列化 | serde + serde_json | Rust标准序列化库 |
| 文件上传 | actix-multipart | Actix-web官方multipart支持 |
| S3客户端 | rust-s3 (crate: rust-s3) | Rust S3兼容客户端，支持MinIO |
| UUID生成 | uuid | 文件重命名、唯一标识 |
| 日期处理 | chrono | 日期时间处理 |
| 日志 | env_logger + log | 开发和运行日志 |
| Excel处理 | calamine + rust_xlsxwriter | Excel读写（批量导入导出） |

## 3. 核心数据模型

> **数据模型以《数据库与数据模型确定版》为准**，本节保持同步。

### 3.1 身体部位字典表 (body_parts)

每个部位对应一个 MinIO 桶。新增部位时程序自动创建对应桶。

```sql
CREATE TABLE body_parts (
    id              SERIAL PRIMARY KEY,
    part_code       VARCHAR(50) UNIQUE NOT NULL,     -- 部位编码（英文，如 knee）
    part_name       VARCHAR(100) NOT NULL,           -- 部位中文名称（如 膝关节）
    bucket_name     VARCHAR(100) UNIQUE NOT NULL,    -- MinIO桶名称（如 medical-knee）
    is_active       BOOLEAN NOT NULL DEFAULT TRUE,   -- 是否启用
    sort_order      INTEGER NOT NULL DEFAULT 0,      -- 排序顺序
    description     TEXT NOT NULL DEFAULT '',         -- 描述
    created_at      TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_body_parts_active ON body_parts(is_active);
```

**初始数据**：


### 3.3 医学记录主表 (medical_records)

每条记录关联一个医院和一个部位（外键）。文件通过 `files` 表关联。

```sql
CREATE TABLE medical_records (
    id              SERIAL PRIMARY KEY,
    record_no       VARCHAR(50) UNIQUE NOT NULL,     -- 记录编号（MR-2026-000001）
    year            INTEGER NOT NULL,                -- 年度
    month           SMALLINT NOT NULL CHECK (month BETWEEN 1 AND 12),
    day             SMALLINT NOT NULL CHECK (day BETWEEN 1 AND 31),
    record_date     DATE NOT NULL,                   -- 完整日期（year+month+day）
    hospital_id     INTEGER NOT NULL REFERENCES hospitals(id),    -- 医院（外键）
    body_part_id    INTEGER NOT NULL REFERENCES body_parts(id),   -- 部位（外键 → 决定MinIO桶）
    description     TEXT NOT NULL DEFAULT '',         -- 描述/备注
    created_at      TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_records_date ON medical_records(record_date);
CREATE INDEX idx_records_hospital ON medical_records(hospital_id);
CREATE INDEX idx_records_body_part ON medical_records(body_part_id);
CREATE INDEX idx_records_year_month ON medical_records(year, month);
CREATE INDEX idx_records_record_no ON medical_records(record_no);
```

对应Rust结构体（Diesel）：
```rust
use chrono::{NaiveDate, NaiveDateTime};

    pub id: i32,
    pub record_no: String,

    pub body_part_name: String,      // JOIN body_parts

}
```

### 3.4 文件元数据表 (files)

每个文件对应一条记录，存储 MinIO 中的定位信息。

```sql
CREATE TABLE files (
    id              SERIAL PRIMARY KEY,
    record_id       INTEGER NOT NULL REFERENCES medical_records(id) ON DELETE CASCADE,
    file_type       VARCHAR(20) NOT NULL CHECK (file_type IN ('image', 'model', 'annotation', 'report')),
    original_name   VARCHAR(500) NOT NULL,           -- 原始文件名
    storage_name    VARCHAR(500) NOT NULL,           -- 存储文件名（UUID.ext）
    bucket_name     VARCHAR(100) NOT NULL,           -- MinIO桶名（冗余，来自 body_parts.bucket_name）
    object_key      VARCHAR(1000) NOT NULL,          -- MinIO对象键
    file_size       BIGINT NOT NULL DEFAULT 0,       -- 文件大小（字节）
    file_extension  VARCHAR(20) NOT NULL DEFAULT '', -- 文件扩展名
    upload_time     TIMESTAMP NOT NULL DEFAULT NOW() -- 上传时间
);

CREATE INDEX idx_files_record_id ON files(record_id);
CREATE INDEX idx_files_file_type ON files(file_type);
CREATE INDEX idx_files_bucket ON files(bucket_name);
```

对应Rust结构体（Diesel）：
```rust
// 查询用

```

### 3.5 MinIO存储结构

**桶（Bucket）设计**：按身体部位分桶，每个部位一个桶

| 桶名称 | 部位 | 存储内容 |
|--------|------|----------|
| `medical-scapula` | 肩胛骨 | 该部位所有业务数据（图像/模型/标注/报告） |
| `medical-elbow` | 肘关节 | 该部位所有业务数据 |
| `medical-pelvis` | 骨盆 | 该部位所有业务数据 |
| `medical-knee` | 膝关节 | 该部位所有业务数据 |
| `medical-ankle` | 足踝 | 该部位所有业务数据 |
| `medical-hand` | 手掌 | 该部位所有业务数据 |

> **部位-桶映射表**：在数据库 `body_parts` 表中维护 `bucket_name` 字段。新增部位时程序自动创建对应桶。

**桶内按业务类型组织**，每个桶内有4个业务前缀：

| 前缀 | 业务分类 | 文件格式 |
|------|----------|----------|
| `images/` | 医学图像数据 | .dcm, .nii.gz |
| `models/` | 模型数据 | .stl |
| `annotations/` | 标注数据 | .zip (DICOM zipped), .nii.gz |
| `reports/` | 报告 | .pdf, .docx, .xlsx, .md |

> **注意**：`.nii.gz` 同时出现在"医学图像"和"标注数据"中，系统通过用户选择的业务类型（而非扩展名）决定存入哪个前缀。

**对象键（Object Key）规则**：
```
{业务类型}/{年}/{月}/{记录ID}/{UUID}.{扩展名}

示例（以 medical-knee 桶为例）：
images/2026/01/42/a1b2c3d4-e5f6-7890-abcd-ef1234567890.dcm
images/2026/01/42/b2c3d4e5-f6a7-8901-bcde-f12345678901.nii.gz
models/2026/01/42/c3d4e5f6-a7b8-9012-cdef-123456789012.stl
annotations/2026/02/42/d4e5f6a7-b8c9-0123-defa-234567890123.zip
annotations/2026/02/42/f6a7b8c9-d0e1-2345-fgab-456789012345.nii.gz
reports/2026/01/42/e5f6a7b8-c9d0-1234-efab-345678901234.pdf
```

**MinIO连接配置**（.env）：
```
MINIO_ENDPOINT=http://localhost:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin
MINIO_REGION=us-east-1
```

### 10.1 Actix-web + Diesel r2d2 服务启动

```rust
use actix_web::{web, App, HttpServer, middleware};
use diesel::r2d2::{self, ConnectionManager};
use diesel::PgConnection;
use tera::Tera;

pub type DbPool = r2d2::Pool<ConnectionManager<PgConnection>>;

#[actix_web::main]
async fn main() -> std::io::Result<()> {
    // 初始化日志
    env_logger::init();
    dotenvy::dotenv().ok();
    
    // 初始化Diesel连接池
    let database_url = std::env::var("DATABASE_URL")
        .expect("DATABASE_URL must be set in .env");
    let manager = ConnectionManager::<PgConnection>::new(&database_url);
    let pool: DbPool = r2d2::Pool::builder()
        .max_size(10)
        .build(manager)
        .expect("Failed to create DB pool");
    
    // 初始化Tera模板引擎
    let tera = Tera::new("templates/**/*").expect("Tera init failed");
    
    // 启动HTTP服务
    HttpServer::new(move || {
        App::new()
            .app_data(web::Data::new(pool.clone()))
            .app_data(web::Data::new(tera.clone()))
            .app_data(web::PayloadConfig::default().limit(500 * 1024 * 1024))
            .service(actix_files::Files::new("/static", "static"))
            .configure(routes::init)
    })
    .bind("127.0.0.1:8080")?
    .run()
    .await
}
```

### 10.2 Diesel CRUD 示例

```rust
use diesel::prelude::*;
use crate::schema::{medical_records, hospitals, body_parts};
use crate::models::{MedicalRecord, NewMedicalRecord, MedicalRecordView};
use crate::db::DbPool;

// 查询列表（分页，JOIN医院和部位表）
pub fn list_records(
    pool: &DbPool,
    page: i64,
    per_page: i64,
) -> QueryResult<Vec<MedicalRecord>> {
    let mut conn = pool.get().expect("DB connection failed");
    medical_records::table
        .order(medical_records::id.desc())
        .limit(per_page)
        .offset((page - 1) * per_page)
        .load::<MedicalRecord>(&mut conn)
}

// 插入记录
pub fn create_record(
    pool: &DbPool,
    new_record: &NewMedicalRecord,
) -> QueryResult<MedicalRecord> {
    let mut conn = pool.get().expect("DB connection failed");
    diesel::insert_into(medical_records::table)
        .values(new_record)
        .get_result::<MedicalRecord>(&mut conn)
}

// 动态条件查询（通过外键ID筛选医院和部位）
pub fn search_records(
    pool: &DbPool,
    year: Option<i32>,
    month: Option<i16>,
    hospital_id: Option<i32>,
    body_part_id: Option<i32>,
) -> QueryResult<Vec<MedicalRecord>> {
    let mut conn = pool.get().expect("DB connection failed");
    let mut query = medical_records::table.into_boxed();
    
    if let Some(y) = year {
        query = query.filter(medical_records::year.eq(y));
    }
    if let Some(m) = month {
        query = query.filter(medical_records::month.eq(m));
    }
    if let Some(h) = hospital_id {
        query = query.filter(medical_records::hospital_id.eq(h));
    }
    if let Some(bp) = body_part_id {
        query = query.filter(medical_records::body_part_id.eq(bp));
    }
    
    query.order(medical_records::id.desc())
        .load::<MedicalRecord>(&mut conn)
}

// 查询记录详情（JOIN获取医院名和部位信息）
pub fn get_record_view(
    pool: &DbPool,
    record_id: i32,
) -> QueryResult<(MedicalRecord, String, String, String)> {
    let mut conn = pool.get().expect("DB connection failed");
    medical_records::table
        .inner_join(hospitals::table)
        .inner_join(body_parts::table)
        .select((
            medical_records::all_columns,
            hospitals::hospital_name,
            body_parts::part_name,
            body_parts::part_code,
        ))
        .filter(medical_records::id.eq(record_id))
        .first(&mut conn)
}
```

### 10.3 MinIO客户端初始化

```rust
use s3::bucket::Bucket;
use s3::creds::Credentials;
use s3::Region;
use std::collections::HashMap;

/// 部位-桶名映射（启动时从数据库body_parts表加载）
pub const BODY_PART_BUCKETS: &[(&str, &str)] = &[
    ("scapula", "medical-scapula"),   // 肩胛骨
    ("elbow",   "medical-elbow"),     // 肘关节
    ("pelvis",  "medical-pelvis"),    // 骨盆
    ("knee",    "medical-knee"),      // 膝关节
    ("ankle",   "medical-ankle"),     // 足踝
    ("hand",    "medical-hand"),      // 手掌
];

/// 业务类型前缀
pub const FILE_TYPE_PREFIXES: &[&str] = &["images", "models", "annotations", "reports"];

pub struct MinioClient {
    /// key = 部位编码（如 "knee"），value = Bucket 实例
    pub buckets: HashMap<String, Bucket>,
    region: Region,
    credentials: Credentials,
}

impl MinioClient {
    pub fn new() -> Self {
        let endpoint = std::env::var("MINIO_ENDPOINT")
            .unwrap_or_else(|_| "http://localhost:9000".to_string());
        let access_key = std::env::var("MINIO_ACCESS_KEY")
            .unwrap_or_else(|_| "minioadmin".to_string());
        let secret_key = std::env::var("MINIO_SECRET_KEY")
            .unwrap_or_else(|_| "minioadmin".to_string());
        
        let region = Region::Custom {
            region: "us-east-1".to_string(),
            endpoint: endpoint.clone(),
        };
        let credentials = Credentials::new(
            Some(&access_key), Some(&secret_key), None, None, None
        ).expect("Failed to create MinIO credentials");
        
        let mut buckets = HashMap::new();
        for (part_code, bucket_name) in BODY_PART_BUCKETS {
            let bucket = Bucket::new(bucket_name, region.clone(), credentials.clone())
                .expect("Failed to create bucket")
                .with_path_style();  // MinIO需要path style
            buckets.insert(part_code.to_string(), bucket);
        }
        
        MinioClient { buckets, region, credentials }
    }
    
    /// 根据部位编码获取对应桶
    pub fn get_bucket(&self, body_part: &str) -> Option<&Bucket> {
        self.buckets.get(body_part)
    }
    
    /// 动态添加新部位桶（新增部位时调用）
    pub fn add_bucket(&mut self, part_code: &str, bucket_name: &str) {
        let bucket = Bucket::new(bucket_name, self.region.clone(), self.credentials.clone())
            .expect("Failed to create bucket")
            .with_path_style();
        self.buckets.insert(part_code.to_string(), bucket);
    }
    
    /// 构建对象键：{业务类型}/{年}/{月}/{记录ID}/{uuid}.{ext}
    pub fn build_object_key(file_type: &str, record_id: i32, uuid_name: &str) -> String {
        let now = chrono::Local::now();
        format!("{}/{}/{}/{}/{}",
            file_type, now.format("%Y"), now.format("%m"), record_id, uuid_name)
    }
}
```

### 10.4 文件上传处理（MinIO）

```rust
use actix_multipart::Multipart;
use futures_util::StreamExt;
use uuid::Uuid;

async fn upload_file(
    mut payload: Multipart,
    pool: web::Data<DbPool>,
    minio: web::Data<MinioClient>,
    path: web::Path<i32>,           // record_id
    form_data: web::Query<UploadParams>,  // file_type 由前端传入
) -> Result<HttpResponse, Error> {
    let record_id = *path;
    
    // 从数据库查询该记录 + JOIN body_parts 获取部位编码 → 确定桶
    let (record, _hospital_name, _part_name, part_code) = get_record_view(&pool, record_id)?;
    let bucket = minio.get_bucket(&part_code) // part_code 如 "knee"
        .ok_or_else(|| actix_web::error::ErrorBadRequest("未知部位"))?;
    let bucket_name = bucket.name().to_string();
    
    while let Some(Ok(mut field)) = payload.next().await {
        let content_disposition = field.content_disposition();
        let filename = content_disposition.get_filename().unwrap_or("unknown");
        let extension = get_full_extension(filename); // 支持 .nii.gz 双扩展名
        let file_type = &form_data.file_type; // image/model/annotation/report（前端表单传入）
        
        // 收集文件内容
        let mut file_bytes = Vec::new();
        while let Some(Ok(chunk)) = field.next().await {
            file_bytes.extend_from_slice(&chunk);
        }
        let file_size = file_bytes.len() as i64;
        
        // 生成对象键：{业务类型}/{年}/{月}/{记录ID}/{uuid}.{ext}
        let storage_name = format!("{}.{}", Uuid::new_v4(), extension);
        let object_key = MinioClient::build_object_key(file_type, record_id, &storage_name);
        
        // 上传到对应部位的桶
        bucket.put_object(&object_key, &file_bytes).await
            .map_err(|e| actix_web::error::ErrorInternalServerError(e))?;
        
        // Diesel写入文件元数据
        let new_file = NewFileRecord {
            record_id,
            file_type,
            original_name: filename,
            storage_name: &storage_name,
            bucket_name: &bucket_name,
            object_key: &object_key,
            file_size,
            file_extension: &extension,
        };
        // diesel::insert_into(files::table).values(&new_file)...
    }
    Ok(HttpResponse::Found()
        .append_header(("Location", format!("/records/{}", record_id)))
        .finish())
}
```

## 11. 注意事项

1. **文件安全**：上传文件使用UUID重命名，防止文件名冲突和路径注入；MinIO桶策略设为私有
2. **大文件**：DICOM文件可能较大，小文件直接内存上传，大文件（>100MB）考虑分片上传（S3 multipart upload）。配置Actix payload限制500MB
3. **Rust编译**：首次编译较慢（拉取依赖+编译），后续增量编译很快。开发用 `cargo run`，发布用 `cargo build --release`
4. **数据库**：PostgreSQL需要单独安装和运行服务。使用 `pg_dump` 定期备份
5. **MinIO**：需要单独部署MinIO服务。数据存储在MinIO的data目录，支持纠删码保障数据可靠性
6. **Diesel迁移**：所有表结构变更通过 `diesel migration` 管理，保证版本可追溯
7. **编码**：所有字符串统一UTF-8，Rust原生支持，PostgreSQL默认UTF-8
8. **错误处理**：Rust强制处理所有错误，使用Result和?操作符传播错误
9. **并发安全**：Diesel r2d2连接池线程安全，Actix-web的actor模型保障并发安全，MinIO原生支持高并发
10. **前提条件**：开发机需安装Rust工具链（rustup）+ PostgreSQL 15+ + diesel_cli + MinIO Server

```powershell
  -v D:\minio-data:/data \
  minio/minio server /data --console-address ":9001"

# 方式3：直接运行（Windows下载exe后）
minio.exe server D:\minio-data --console-address ":9001"

# 访问MinIO管理控制台
# http://localhost:9001
# 用户名: minioadmin
# 密码: minioadmin

# API端点
# http://localhost:9000
```
