# 模型展示页重构架构方案

> 范围：**仅重构模型展示页**，其他页面保持 Tera 模板不变  
> 方案：Vite + Vue 3 + @kitware/vtk.js (npm 完整版) 独立 SPA  
> 日期：2026-02-27

---

## 一、改动范围

| 页面 | 方案 | 变化 |
|---|---|---|
| 首页（记录列表） | Tera 模板 `index.html` | **不动** |
| 新建记录 | Tera 模板 `record_new.html` | **不动** |
| 记录详情 | Tera 模板 `record_detail.html` | **不动** |
| 编辑记录 | Tera 模板 `record_edit.html` | **不动** |
| **模型展示** | ~~model_gallery.html / model_viewer.html~~ | **替换为 Vue SPA** |

**砍掉的组件**：
- `vtk-renderer/` Python 服务 — 不再需要后端渲染
- `templates/model_gallery.html` — 后端推帧方案
- `templates/model_viewer.html` — 旧版裁剪 vtk.js 方案
- `static/js/vtk.js` (2.3MB 裁剪版) — npm 完整版替代
- WebSocket 推帧机制 — 前端直接 GPU 渲染

## 三、Rust 后端改动（最小化）

### 新增 2 个 JSON API

```rust
// 1. 获取某记录下的模型文件列表
// GET /api/records/{record_no}/models
// 返回：
{
  "record_no": "MR-2026-000001",
  "files": [
    {
      "id": 16,
      "original_name": "scapula_left.stl",
      "file_size": 453184,
      "file_size_str": "442.6 KB"
    },
    ...
  ]
}

// 2. 获取文件二进制数据（已有 view_model_file，改路径即可）
// GET /api/files/{file_id}/data
// 返回：application/octet-stream (STL 字节流)
```

### 入口跳转

记录详情页 `record_detail.html` 的「模型浏览」按钮链接改为：

```
/viewer/?record_no=MR-2026-000001
```

### 静态托管

```rust
// main.rs 中新增，托管 Vue 构建产物
cfg.service(Files::new("/viewer", "./medical-web/dist").index_file("index.html"));
```

### CORS（开发期）

仅开发模式下需要，生产时同源无需 CORS。

## 四、前端项目结构

```
medical-web/                    ← 新建，仅模型展示页
├── index.html                  ← SPA 入口
├── package.json
├── vite.config.ts
├── tsconfig.json
└── src/
    ├── main.ts                 ← Vue 挂载
    ├── App.vue                 ← 根组件（单页面，无路由）
    ├── api/
    │   └── files.ts            ← fetch 封装（仅 2 个接口）
    ├── components/
    │   ├── FileList.vue        ← 左侧文件列表
    │   ├── Topbar.vue          ← 顶部工具栏（返回/文件名/提示）
    │   └── VtkRenderer.vue     ← vtk.js 渲染区（核心）
    └── vtk/
        ├── pipeline.ts         ← 渲染管线：Renderer + RenderWindow + Interactor
        ├── lights.ts           ← 三点光源配置
        ├── material.ts         ← 骨骼材质参数
        └── loader.ts           ← STL 加载 + Actor 替换（不重建管线）
```

**说明**：
- 无需 vue-router — 整个 SPA 只有一个页面
- 无需 axios — 只有 2 个 fetch 调用
- URL 参数 `?record_no=xxx` 传入记录编号

## 五、VTK 渲染管线

```
URL 参数 record_no
    │
    ▼
fetch /api/records/:no/models → 文件列表
    │
    ▼
初始化管线（只执行一次）:
    vtkRenderer + vtkRenderWindow + vtkOpenGLRenderWindow
    + vtkRenderWindowInteractor + TrackballCamera
    + 三点光源 + SSAO Pass
    │
    ▼
用户点击文件项
    │
    ▼
fetch /api/files/:id/data → ArrayBuffer
    │
    ▼
vtkSTLReader.parseAsArrayBuffer(buf)
    │
    ▼
mapper.setInputData(reader.getOutputData())  ← 复用 mapper/actor
    │
    ▼
renderer.resetCamera() → renderWindow.render()
    │
    ▼
用户交互 → TrackballCamera 直驱 GPU（零延迟）
```

### 渲染配置

```typescript
// ── 三点光源 ──
keyLight:  { position: [1, 1, 1],     intensity: 1.0 }
fillLight: { position: [-1, 0.5, 0],  intensity: 0.6 }
backLight: { position: [0, -1, -1],   intensity: 0.3 }

// ── SSAO (环境光遮蔽) ──
ssaoPass:  { radius: 0.5, bias: 0.05, kernelSize: 32 }

// ── 材质 ──
actor.property: {
  color:         [0.85, 0.79, 0.68],  // 骨骼米色
  ambient:       0.2,
  diffuse:       0.8,
  specular:      0.3,
  specularPower: 20
}

// ── 背景 ──
renderer.background: [0.059, 0.090, 0.165]  // #0f172a
```

## 七、开发与部署

### 开发

```
终端 1: cd medical-server && cargo run      # Rust :8080（现有页面 + 新 API）
终端 2: cd medical-web && npm run dev        # Vite :5173（仅模型页开发）
```

vite.config.ts:
```typescript
export default defineConfig({
  base: '/viewer/',
  server: {
    proxy: {
      '/api': 'http://localhost:8080',
    }
  }
})
```

### 生产

```bash
cd medical-web && npm run build    # 输出到 dist/
```

Rust 直接托管 `dist/` 目录到 `/viewer/` 路径下，单进程部署，无需 Nginx。

## 九、对比

| | 当前方案 | 新方案 |
|---|---|---|
| 渲染位置 | Python 后端 CPU/GPU | 浏览器 WebGL2 GPU |
| 交互延迟 | 40-100ms（网络往返） | **0ms**（本地 GPU 直渲染） |
| 渲染质量 | VTK 原生（好） | vtk.js + SSAO + 三点光源（好） |
| 依赖服务 | Rust + Python + WebSocket | **仅 Rust** |
| 改动范围 | — | 仅模型展示页，其他页面不动 |
